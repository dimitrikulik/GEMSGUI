1. GEM2MT Module Operation

GEM2MT module is a tool for automation of simple reactive mass transport simulations coupled with GEM calculation of equilibrium states in spatially distributed nodes (boxes, volumes) over time steps. This module performs simulations of the evolution of fluid-rock interaction reactive transport systems, where the fluid is moved sequentially or simultaneously from one box to the other. This type of simulation represents a simple reactive mass transport model, which requires definition of fluxes between boxes of certain solid-water mass or volume ratios, alternatively porosity and permeability. This model can be used to simulate an open system, and permits to test one-dimensional reactive transport with complex chemical reactions, that may be later used as a benchmark for the coupling of GEMS3K with more complex reactive transport models.

The GEM2MT module can simulate reactive transport in three main modes:

  1. Sequential reactor chain and waves (S);
  2. Flow-through box-flux sequence (F, B);
  3. One-dimensional reactive transport with advection/dispersion/diffusion (A,D,W)

1-D advection-diffusion numerical reactive transport simuations can be performed, as well as the evolution of various sequential and flow-trough reactor sequences or generic box-flux (reservoir) dynamics can be modeled. In addition, I/O files for standalone GEMS3K calculations can be exported for multiple initial node recipes (in the T mode). Changes of species concentrations or amounts of phases through the node profile with time can be graphically monitored at each time step (wave) or logged to files in VTK format.

Below, some relevant features of main types of reactive transport simulations are summarized.


1.1. Sequential reactor chains and waves (S mode)

Such simulations are useful for predicting how the rock alteration zones develop upon a progressive infiltration of the fluid supplied from a constant source (e.g. a large fracture). This is a good model for some steady-state irreversible processes such as weathering, infiltration metasomatosis, evaporation of seawater etc.

The simplest case is when there is only one "reactor" (box) at given temperature T and pressure P, in which initially only rock of known bulk composition is present at a prescribed initial mass. At step 0, the fluid of defined composition is added to the "rock" at a given fluid-rock mass ratio and left to equilibrate, then the aqueous composition/ speciation and the phase assemblage and composition of the reacted rock is explored (sampled). At step 1, the reacted fluid in the box is replaced by the "fresh" fluid at the same fluid-rock mass ratio, and equilibrated again. The loop is repeated many times, allowing the user to follow how the phase composition of altered rock changes along the way. Note that this type of simulation can be done in the Process module of GEM-Selektor as the leaching-type "flow-through" reactor 'R' mode. 

The GEM2MT module permits to set up and run a more natural and flexible sequential scheme consisting of many "reactors" (boxes) arranged in a 1-D chain, where the fluid (water) moves from the previous to the next box. The boxes may, in principle, have different temperatures and pressures, if desired. The constant composition "fresh" fluid enters the very first box at a prescribed fluid-rock mass ratio and reacts there. The equilibrated fluid is taken to the next box (at prescribed fluid-rock ratio) and reacts there, and so on until the last box is reacted. This completes a so-called "wave".

Any number of SR "waves" can be performed. At the beginning of second wave, the "fresh" fluid is put into the first box with the reacted rock composition kept after the first "wave". After equilibration, the fluid is taken into the second box to react with the rock already reacted at the first "wave", and so on. After many "waves", the formation and propagation of rock alteration zones can be followed by plotting amounts of phases in all boxes after each "wave".


1.2. Flow-through box-flux sequences (F, B modes)

In the F mode, the 1-D reactive transport column is represented by many equal-size boxes that may have different temperatures and pressures, if desired. The first box must be coded as a constant-flux (Cauchy) source (3), and the last box - as a constant-flux sink (-3); all the boxes in-between are normal local-equilibrium systems (1). Equal zero-order fluxes of aqueous fluid "aq" (not too high!) should be set between boxes 0 and 1, 1 and 2, 2 and 3, ... until nC-2 and nC-1 (sink).

Composition of the "aq" MPG (mobile phase group) should be defined in the PGT table (units in UMGP column) in the "Fluxes" page of GEM2MT window. A column in PGT table defines quantities of phases moved from the source box in one mole of the flux. If there is another MPG defined, e.g. "gas", then two-phase transport between boxes can be arranged. 

At a unit time step, a given mass of "aq" flux (reacted aqueous fluid) is moved into the next box replacing there the equivalent mass of fluid that moves to the next box and so on simultaneously (fresh fluid from the constant-flux source moves from box 0 into box 1, and the same mass of reacted fluid moves away from the last box). In fact, this simultaneous transport is consistently performed by the built-in integrator of the ODE system of mass balance equations for all elements in all nodes.

The mass of fluid moved by the flux between nodes n and n-1 is defined by the flux composition, flux rate and flux order. The simplest zero-order (constant) flux moves 1 mol of its composition times the flux rate (set in the FDLf table on "Fluxes" page). The rate 0.1 means that 0.1 mol of the flux composition moves per step (unit time). For the 1st order flux, the rate 0.1 means that 0.1 mol of flux composition times the mass of its source MPG (in kg) is moved per step. Such a flux will automatically decrease if the mass of its source phase(s) in the source box decreases, and vice versa. 

After the transport step, the equilibrium speciation is computed in all nodes by the GEM algorithm, and the next iteration of transport occurs followed by the next equilibration in all boxes, and so on, until the maximum number of steps or the total time of simulation is reached. After each equilibration, the masses, volumes or concentrations of interest (both in aqueous and solid parts) are sampled and plotted.

The F mode simulation with equal zero-order fluxes is, in fact, analogous to 1-D pure advection transport, except that the time step is set to 1, the tracer dispersion is ignored, and the hydraulic consistency criteria are not applied.

The B mode operates in a similar way except the combination of boxes and fluxes may be arbitrary (not only chain-like), the boxes can have different size, and the fluxes may be of different order and rate. This setup is suitable e.g. for simulating geochemical cycles or redox-sedimentation dynamics in water columns. The transport step is performed by using the ODE integrator of the system of mass balance equations for all elements in all boxes.

Theoretical background of the mass-balance integration in a reservoir box-flux "megasystem"can be found e.g. in [Lerman A., Mackenzie F.T., Garrels R.M. (1975), Modeling of geochemical cycles: Phosphorus as an example, Geol. Soc. Amer. Memoir 142, p. 205-218].  


1.3. One-dimensional reactive transport (A, C, W, D modes)

The GEM2MT module allows the user to set up and run simplified 1-D simulations of reactive transport in porous media using the finite-difference advection-dispersion algorithm (classical A mode and implicit Crank-Nicolson C mode TBD) for advection-dominated cases. For the diffusion-dominated transport, the random-walk algorithm (W mode, also accounts for advection) can be used; the pure diffusion transport method (D mode) is under construction in this version of GEM-Selektor. Theoretical background for all these transport modelling methods can be found in numerous textbooks.

The non-iterative coupling of transport and chemistry is implemented in all cases (so-called "operator splitting" approach): the mass transport model integration over all nodes alternates with GEM calculation of equilibrium states in all nodes after each time step, until the maximum time or number of time steps is reached. The profiles of amounts of phases or aqueous species of interest can be plotted after each time step (or exported to VTK files).

By no means this simple implementation of 1-D reactive transport is designed to compete with popular coupled codes such as PFLOTRAN, OpenGeoSys or CRUNCH. Rather, the purpose of A, C, W modes in GEM2MT is education and testing of reactive system behavior before performing heavy computations of 2-D or 3-D cases using coupled RT codes on parallel high-performance computer hardware.

One particular interest is to compare the megasystem evolution between 1-D box-flux (mode F, B) models with ODE integration and explicit fluxes, and more rigorous hydraulic advection-dispersion (diffusion) models with concentration gradients (of total dissolved elements or aqueous species) solved by PDE intergration (A,C modes) or stochastic simulation of Brownian movement of particles (W mode).

The main difference between the box-flux models and the hydraulic advection-dispersion-diffusion models is the choice of transport parameters (defined initial state for all boxes or nodes is required in all cases). While in B or F modes, the zero- or 1st order fluxes must be set by MPG composition, flux rate or rate constant, in A,C,W modes, initial transport parameters in all nodes must be defined. These include porosity, permeability, advection velocity, dispersivity, tortuosity and diffusivity (common or individual diffusion coefficients).

These parameters can be conveniently set upon initialization of the RT problem using the initialization script. If needed, some parameters in some nodes may change in the process (e.g. due to mineral dissolution or precipitation, porosity and permeability may change), which has to be programmed in the last part of initialization script.  



2. GEM2MT Create/Remake Wizard

The GEM2MT simulator is a tool for automation of simple reactive mass transport models coupled with GEM calculation of equilibrium states in spatially distributed nodes (boxes, volumes) over time steps.

It is possible to perform 1D advection-diffusion numerical finite-difference modeling, as well as the box-flux (reservoir) dynamics.

Changes of species concentrations or amounts of phases through the node profile with time can be graphically monitored and logged to disk files.

The GEM2MT simulator can be configured by choosing its mode of operation and setting up the number of nodes (boxes, volumes).

You may also have to type math scripts that define how to distribute initial system compositions over nodes, and which data to sample from nodes for graphic presentation or export to text files.

This wizard helps you configure and set-up the RMT simulation step-by-step.

Possible GEM2MT tasks fall into three groups:
  • Box-flux (S,F,B) simulations, where the transport is defined in terms of fluxes of mobile phase groups (MPG) between boxes or reactors.
  • 1-D reactive transport simulations using numerical advection-dispersion finite-differences (A), Crank-Nicolson (C), advection-diffusion finite-volumes (D) or advection-diffusion random-walk (W) algorithms.
  • Preparation (T) of initial chemical systems and exporting them into GEMS3K input files for external coupled RMT codes.

S     Sequential reactors chain with waves (moving fluids or solids between reactors)

This flexible scheme consists of many "reactors" (boxes) arranged into a 1-D chain, where at each step, the whole fluid phase is moved (copied) from the previous into the next box. The constant composition "fresh" fluid enters the very first box with a prescribed fluid-rock mass ratio and reacts there. The equilibrated fluid is taken to the next box (at prescribed fluid-rock ratio, replacing the fluid left from the previous "wave") and reacts there, and so on until the last box is reacted, thus completing the so-called "wave". Any desired number of "waves" can be done. At the beginning of second wave, the "fresh" fluid replaces the old fluid phase in the first box. After equilibration, the fluid is taken into the second box to react, and so on. From "wave" to "wave", the propagation of rock alteration zones can be followed by plotting amounts of phases in all boxes. A variant of S mode allows to move from box to box all solids instead of the fluid phase, like in a dialysis bag (to follow the fluid evolution).

2.2. GEM2MT Wizard Step 2: Dimensions for the RT model and initial chemical systems

Set here the number of local-equilibrium nodes (boxes, volumes) for the transport model (except T mode), and the number of system definitions for initialization of nodes (all but T modes) or for export to GEMS3K input files (T mode). You can set also some options related to usage of math scripts and file output of simulation results.

nC defines how many spatially distributed nodes (boxes, volumes) will be considered in the mass transport model. Typically, 10 to 1000 nodes can be used. Exception: in  T mode, nC = nIV must be set.

Tau defines the physical time span for A, D, W, F, B type RT models. The start value and the step value can be set. The unit of measurement is arbitrary (i.e. the user must keep it in mind), although all the transport parameter values (fluxes, velocities) must be consistent with the chosen time unit (e.g. second).  

ntM  is the maximum number of time steps (at least one; default 1000, negative- unlimited). GEM2MT simulation will stop when this number is reached or upon the user's cancel command. The "real" time scale can be defined later in the GEM2MT window.

nIV is the number of initial chemical system definitions that are needed for the initial distribution into all nodes to define the time-zero state of the reactive mass transport problem. This distribution must be set in the "DiCp" vector, which can be filled out upon an execution of the math script for node initialization. In many cases, this script can be created by this wizard, then perhaps modified by the user before the GEM2MT run.

Lb is the number of optional user-defined formula units for setting composition recipes of initial chemical system definitions (on Page 1 of GEM2MT window). If Lb is set to 0 then the tick below should be set on to use the quantities of Independent Components instead (both ways can be used together, in which case they will add up into the bulk composition vectors of initial systems). Typical formula units are oxides ("H2O", "CO2", ... ) or other compounds, or more complex stoichiometries like bulk rock compositions. In GEM2MT module, the length of formula unit is limited to 40 characters.

Smart initial approximation for GEM calculations for each node at every time step can be activated to save on the computational time. Deactivate this mode if error messages appear or to check whether the same results are obtained with the automatic initial approximation (takes longer).

If options for math scripts are not selected then the respective wizard pages will not be accessible, and the text editor fields on GEM2MT window pages will not be visible. However, in some simulation modes, the scripts are mandatory and will be created irrespective of the user's selection.


2.3. GEM2MT Wizard Step 3: Settings for the GEMS3K data exchange interface

These settings are important for the additional T mode for preparing the GEMS3K .dch, .ipm and .dbr input files for external coupled codes, and may also be useful for certain built-in reactive transport modes. Settings are arranged in three groups.

(1) GEMS3K memory and file data exchange

Select ICs, DCs and phases to be exchanged via DATABR (take all, if unchecked): Having this option unchecked is recommended, which will take all components and phases into RT data exchange bridge.

Write IPM, DCH and DBR files in binary mode (in text mode, if unchecked): Text mode (unchecked) is default and recommended for keeping the files human-readable.

Do not write data items that contain only default values (text mode only): If this option is checked, only the data entries that contain non-default values will be written into data exchange files, while those containing only default values will be skipped. This produces compact files that are easy to read and modify in text editor (especially DBR files), but may potentially become incompatible with future GEMS3K versions if some default values will change. Leave unchecked if in doubt.

Write .dch, .ipm, .dbr files with comments for all data entries (text mode only): This is the "verbose" mode for education purposes. If this option is checked, a comment line will appear before each data entry, explaining what data it contains (similar to GEM-Selektor tooltips on data objects). May be safely unchecked by experienced users.

(2) Automatic setup of GEMS3K lookup arrays with thermodynamic data

This setup is used for creating lookup arrays for thermodynamic properties of DCs (dependent components) with regular T (temperature) and P (pressure) grid. This applies to standard Gibbs energies go, molar volumes Vo, optionally ehthalpies ho, entropies So and heat capacities Cpo (the optional arrays will be created if the respective properties are present in the parent SysEq records).

Default is one pressure value (1 bar) and one temperature value (25 oC), in which case no interpolation of thermodynamic data can be performed, and all RT calculations will be limited to 1 bar 25 oC only. Another single value of T or P can be set by entering equal number in "Minimal" and "Maximal" fields, while leaving 0 in "Step" field. RT calculations will be performed if the actual T and P values in nodes deviate from the grid T, P values plus-minus the respective "Tolerance".

To perform RT simulations with variable T and/or P, the lookup arrays must be created by setting "Maximal" T or P greater than "Minimal", and setting "Step" to 5 - 10 C for temperature and 10 - 50 bar for pressure. Usually, good interpolation results require 30-40 grid points in a large temperature range and 10-20 points in a large pressure range. Leaving one of "Step" values zero means that simulations will be performed at constant value of this parameter and variable values of another. For a given node, the interpolation of thermodynamic data in the built-in GEMS3K code will be done if the actual T, P values for the node differ from the closest grid point T, P pair by more than "Tolerance".

(3) T and P grid values for lookup arrays

Enter pressure and temperature grid values instead of automatic setup. If this option is activated, the automatic setup of T, P grid is disabled. Set the required numbers of T and P grid values in spin boxes, and enter manually or copy-paste irregular T and P sequences for the lookup grid into the table appearing in the next Step 3b page of this wizard.

Disable data interpolation between grid T values (or P values), use data for TP pairs instead. Activation of this option disables the data interpolation over the lookup T, P grid array. In this case, equal number of T and P points must be entered, and the T, P values must be entered pairwise into the respective columns in the next Step 3b page of this wizard. RT calculations can then only be performed if T, P values for a given node correspond to one of the given T, P pairs (plus-minus "Tolerance").


2.4. GEM2MT Wizard Step 4: Settings for mass fluxes in reactor/box models (S, F, B modes)

This wizard page appears in S, F and B modes only and completes the setup of  RT simulations based upon reactors (boxes) connected with defined fluxes of aqueous fluid (S mode) or MPG (mobile phase groups in F, B modes). The number of boxes nC should have already been defined at Step 2. nC tells how many equilibrating reactors (boxes) will be considered in the megasystem.

At each time step, the transport of matter between boxes is performed by fluxes of mobile phase groups (MPG). There can be MPG fluxes of three types: normal, source, and sink.

A normal flux is defined by the index of its source box, where the flux begins; the index of its destination box, where the flux ends; the index of MPG definition; the flux order (0, 1, 2, 3), and the flux rate constant. The flux order default is 0 (constant flux in units mass/timestep).

Possible flux orders: 0 - Zero-order (absolute) flux, with velocity v in mass/time units;  1 - First-order to source box mass flux, v in 1/time/kg units; 2 - Second-order to source and sink MPG mass flux, v in 1/time/kg2 units, proportional to product of masses in both boxes; 3 - First-order flux to MPG mass in receiving box, v in 1/time/kg units. 

A source flux is defined by its bulk composition (amounts of independent components), which can be entered in the GEM2MT window if nSFD > 0 is set below on this wizard page; the index of its destination box; the flux order (0, 1, 3); and the flux rate constant.

A sink flux is defined by the index of its source box; the index of MPG definition; the flux order (0, 1, 2); and the flux rate constant.

All nFD fluxes should be set in the list accessible on Fluxes page of GEM2MT module window. nSFD bulk compositions of source fluxes should be provided in the table for source fluxes at the same page. All this has to be done by the user in the B mode. However, in S and F modes, nearly the whole setup of fluxes will be done automatically. The user must only select which phases to include in a single MPG (using check-boxes below). Optionally, the user can set nSFD to 1 which will define a source flux into the first reactor (and enter its bulk composition later on), and then proceed to the next wizard step.

The settings are  organized in two groups.

(1) Dimensions of MPG fluxes:

nPG - Number of mobile phase groups (MPG): set it only for the B mode (will be set automatically to 1 in S and F modes)

nFD - Total number of MPG flux definitions: set it only for the B mode (will be set automatically in S and F modes)

nSFD - Number of source flux definitions: optional to set in B, S and F modes. In S and F modes, can be 0 or 1; if set to 0 then the first reactor (index 0) will be treated as constant source of MPG transported into the second reactor; otherwise, the composition of the source flux at a given rate will be added to bulk composition of 0-th reactor at every time step.

(2) Only S and F mode (sequential and flow-through reactors):

Move aqueous (&aq) phase: this is default for F and S - mode reactor setups, the moving MPG will consist of aqueous phase only.

Move gas/fluid/plasma (&gas) phase(s): Composition of all gaseous phases together can be moved from one reactor to the next, also as an addition to aqueous phase composition.

Move all &solids together ('dialysis bag'): Alternatively to two above options, the bulk composition of all solid phases will be moved from reactor to reactor.

In S and F modes, the outgoing flux from the last reactor is automatically defined as a sink flux.


2.5. GEM2MT Wizard Step 5: Auto-script for nodes/boxes initialization and time change

This page configures the automatic setup of initial properties of nodes / boxes in the megasystem by execution of the automatically created script that appears on the right side.  After the wizard is completed, In the GEM2MT window, the initial script can be modified and extended; just click 'Next >' to keep the old script with user's modifications.

The script has two sections - one for the initialisation of the RT problem, another for execution at regular time steps. The first block (before 'else') for the initialization of nodes / boxes using bulk compositions of initial systems will be executed once at time step 0, while the second block (if not commented out) will be executed at every subsequent time step. The initialization script can save a lot of time, especially when there are many nodes / boxes to be created.

Checking/unchecking any option will result in an insertion/removal of the respective operator(s) into the script part for nodes initialization. The exact form of operators, as well as the availability of some options, depends on the chosen RT simulation type. Options 'Pressure' and 'Temperature' are normally selected by default in all RT cases, and set for all nodes a constant P and T pair taken from Step 3 of this wizard. Options 'Initial system', 'Node type' and 'MPG flux' will be automatically activated in S or F modes; the respective parts of the initialization script will be set automatically to create a sequential unidirectional chain transport of MPG from node to node.

Pressure P: This operator(s) define(s) what pressures (in bar) will be assigned to what nodes (boxes)

Temperature T: This operator(s) define(s) what temperatures (in oC) will be assigned to what nodes (boxes)

Volume V: This operator(s) define(s) volumes (in m3) to be assigned to nodes (boxes)

Initial system: Defines (index of) what initial system to copy into the respective node;

Node type: This operator defines the type of the node (box):

0:   normal node;
      Boundary condition nodes:
1:   Dirichlet source (constant composition source);
-1:  Dirichlet sink;
2:   Neumann source (constant gradient source);
-2:  Neumann sink;
3:   Cauchy source (constant flux source);
-3:  Cauchy sink;
4:    Input time-depended function.

MPG flux: This operator(s) defines the indexes of incoming and outgoing MPG fluxes to the node;

Initial mass: Defines the initial mass (in kg) of each node (box).

Options available for 'true' 1-D mass transport models (A, D, W) only:

Initial node volume: define(s) total volumes (in m3) to be assigned to nodes for use in transport model only;

Advection velocity: defines advection velocities (in porous medium) to be assigned to nodes (in m/s);

Initial node porosity: defines Effective (actual) porosity ( in scale from 0 to 1) to be assigned to nodes;

Init. eff. permeability: defines effective permeability (in m2) to be assigned to nodes;

Init. media dispersivity: defines hydraulic longitudinal dispersivity (m2/s) to be assigned to nodes;

Init. general diffusivity: defines general diffusivity of disolved matter (m2/s) to be assigned to nodes;

Init. tortuosity factor: defines tortuosity factor of porous medium (dimensionless) to be assigned to nodes.

Optional:

Number of elements in the script work array qpt (0 to 20): can be set for user-defined additions to this script if work cells must be used in calculation of some intermediate values.


2.6. GEM2MT Wizard Step 6: Script for graphical monitoring of simulations

To monitor (graphically or in a table) some changes in the nodes / boxes profile upon time steps, the script can be automatically created after setting the dimensions and selecting the node properties, components and phases. Optionally, some experimental data can be plotted over the node profile(s). To retain the old monitoring script, just click 'Next >' on this page.

This monitoring script is independent of the VTK flie output that can be configured at the next wizard step.

The graphics dialog will be updated for every time step (or each "wave" in S mode). The simulation can be stopped any time by closing the Graph dialog and clicking "Cancel" on the appearing small dialog. In some modes, the simulation can be resumed later.

Number of plots (columns in the yt table, from 0 to 20) to sample the data from all nodes:  we recommend to set this to 1, 2 or 3 in order to keep the plot easily understandable (all results can be sent to VTK files and any profile can then be shown using Paraview or similar software).
Note: the number of rows in the yt table will be automatically set to nC (total number of nodes / boxes).

Number of rows (points) in the xEt, yEt arrays for plotting experimental data (optional): This needs to be set if you want to plot some experimental data for some step (time point) over the generated curves.

Number of columns (plots) in the  yEt array for plotting experimental data (optional): This needs to be set if you want to plot some experimental data for some step (time point) over the generated curves. The data can be entered or copy-pasted in the GEM2MT window page after this wizard is finished.  

Number of elements in the script work array qpg for graphical monitoring (optional): Enter here an integer (0 to 20) to have access to desired number of work cells (qpg[0], qpg[1], ...) for keeping temporary results in the monitoring script, if necessary.


2.7. GEM2MT Wizard Step 7: Configure output of results to VTK-format files

The graphical monitoring (set at the previous step) can only show the node profiles for selected properties at a current time step, but not a property profile for a selected node over different time steps.

The complete output of RT simulation results can be done in VTK text format files (see http://www.vtk.org) written into a folder with the same name as this GEM2MT record. Each file in that folder corresponds to a time step and contains the values of all selected properties arranged in data vectors corresponding to profiles over all nodes / boxes.

The generated VTK files can be retained and visualized later by using Paraview or similar external software.

Optional:  Select data fields to VTK output (if unchecked, use all DATABR data fields)

If this option is off (default) then all scalars and data for all components and phases available in the last instance of the DATABR node array in GEM2MT will be written into VTK files. Otherwise, you have to do selections from the lists appearing below.

To retain the selections done earlier, just click 'Next >'  to go to the next wizard page.

FDLf[0] [nFD][4] Part of the flux defnition list: flux order {0 1 2 3 }

In this column, the flux order has to be defined. Default is 0 (constant flux in units mass/timestep). Possible values: 0 - Zero-order (absolute) flux, with velocity v in mass/time units;  1 - First-order to source box mass flux, v in 1/time/kg units; 2 - Second-order to source and sink MPG mass flux, v in 1/time/kg2 units, proportional to product of masses in both boxes; 3 - First-order flux to MPG mass in receiving box, v in 1/time/kg units. 
At each time step, the transport of matter between boxes is performed by fluxes of mobile phase groups (MPG). There can be MPG fluxes of three types: normal, source, and sink.

A normal flux is defined by the index of its source box, where the flux begins; the index of its destination box, where the flux ends; the index of MPG definition; the flux order (0, 1, 2, 3), and the flux rate constant.

A source flux is defined by its bulk composition (amounts of independent components), which can be entered in the GEM2MT window if nSFD > 0 is set below on this wizard page; the index of its destination box; the flux order (0, 1, 3); and the flux rate constant.

A sink flux is defined by the index of its source box; the index of MPG definition; the flux order (0, 1, 2); and the flux rate constant.
