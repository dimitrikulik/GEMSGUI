GEM-Selektor version 3

A Gibbs Energy Minimization program package for interactive thermodynamic modeling of heterogeneous aquatic (geo)chemical systems, especially those involving metastability and dispersity of mineral phases, solid solution - aqueous solution equilbria, and adsorption/ion exchange. Includes a built-in (default) thermodynamic database in both thermochemical and reaction formats. Processes of irreversible chemical mass transfer can be simulated from the principles of local and partial equilibrium. A standalone variant of GEMS3 solver - GEMS3K - can be used in coupled reactive transport modeling.

HOW TO START USING GEM-SELEKTOR

To calculate equilibria in a modelling project:

In the upper-left corner of the main window, click on the 'Computation of Equilibria' icon

then select a modelling project from the appearing list and open it, or create a new project.

After opening the project, select a single system from the list on the left side and click on the 'Calculate equilibrium state' toolbar icon . Alternatively, create a new single system using 'Record' 'Create' or 'New(Clone)' menu command or toolbar icon.

To create or work with the Process Simulator, GtDemo or other module, click the respective icon on the leftmost side of main window. To check 'Numerical Settings', click the 'Project' icon.

To close modelling project, click on the 'Database mode' icon in the upper-left corner. This switches the program from 'E' (Calculation of Equilibria) mode to 'D' (Database Management) mode. To open another project, click again at the same place.

To browse, edit or add data in project database:

In the 'D' (Database Management) mode, click on 'DComp', 'ReacDC', 'Phase' or another icon on the leftmost side to open the respective module window, then browse/select a record in the list appearing to the left from the module window. The 'template filter' field above the list can facilitate finding a record in a long list.

Several modules can be kept open simultaneously. To switch between module windows, use a drop-down list above the 'template filter' field.

To access and manage more database modules, use the 'Modules' menu.

To open Help Window (separate from the Main Window):

click in a data object field on the module window page and press F1.

Alternatively, click 'Help' button or the context-sensitive 'Show Help' icon .

Most icons, buttons, and data fields display tooltips on hovering the mouse pointer over them for 1-2 seconds.





ABOUT

GEM-Selektor version 3 (GEMS3) is a Gibbs Energy Minimization program package for interactive thermodynamic modeling of heterogeneous aquatic (geo)chemical systems, especially those involving metastability and dispersity of mineral phases, solid solution - aqueous solution equilbria, and adsorption/ion exchange. Processes of irreversible chemical mass transfer can be simulated from the principles of local and partial equilibrium.

The GEMS3 package includes a built-in (default) thermodynamic database in both thermochemical and reaction formats. 'Kernel' part of this database corresponds to the current edition of the Nagra-PSI database (Hummel et al., 2002). 'Complementary' part consists of the imported SUPCRT92 (Slop98.dat) database, commonly used in hydrothermal geochemistry. There are specific databases that can be downloaded from web sites of their developers (see more in Help documentation).

A standalone variant of GEM solver - GEMS3K - is provided separately for use in coupled reactive transport modeling and related applications. The same numerical kernel (enhanced with the interpreter of GEMS Phase scripts) runs inside of the GEMS3 code.

The GEMS3 code is written in C/C++, using the LGPL version of the Qt4.7 UI framework (http://qt.nokia.com) and the GNU gcc compiler, which makes GEMS3 a cross-platform software for all major Linux, Mac OS X and Windows systems.

The GEMS3K code (the numerical kernel of GEMS3) is a C++ program without any GUI, only with ASCII file I/O, but capable to be compuled and executed also on parallel architectures. GEMS3K uses the public domain JAMA/C++ linear algebra package by Roldan Pozo (http://math.nist.gov/tnt).

OVERVIEW

This chapter is for people who have never used  GEM-Selektor or other Gibbs energy minimisation codes, and, perhaps, have never done geochemical modelling. If you are already familiar with the subject and just try to learn a particular feature or technical information, you can probably skip three sections below.

 

What is GEM-Selektor?

  • GEM-Selektor (GEMS) is a broad-purpose geochemical modelling code. It uses an advanced convex programming method of Gibbs energy minimization (GEM) implemented in an efficient Interior Points Method (IPM) numerical module.

  • Using GEMS, you can compute equilibrium phase assemblage and speciation in a complex chemical system from its total bulk elemental composition at given temperature and pressure (optionally, with some metastability or kinetic constraints).

  • Chemical interactions involving (dispersed) solids, solid solutions, gas mixture, aqueous electrolyte  (non-)electrostatic surface complexation, and ion exchange can be considered simultaneously in the chemical elemental stoichiometry (+ electrical charge) of the system, i.e. without any additional mass balance constraints for ligands or surface sites.

  • GEMS package comes with a built-in write-protected default chemical thermodynamic data base, selected parts of which are automatically copied into your  modelling project data base . There thermodynamic and compositional input data are immediately available for calculation of equilibrium states. In principle, all you have to do is to provide the bulk composition of your system and, possibly, exclude some irrelevant or metastable species and phases. 

  • If necessary, the standard-state thermodynamic data will be automatically converted to temperature/ pressure of interest  before computing the equilibrium state using several techniques appropriate for soilds, gases, aqueous and surface species.  

  • Using GEMS, you can simulate various irreversible mass-transfer processes, such as titrations, mixing, weathering, or sequential reactors, from principles of local and partial equilibrium, controlled by a "process extent" master variable(s) used in the process simulator math scripts. Results of such "process simulations" can easily be tabulated, plotted (also during calculations), or printed into text files.

  • In addition to the common "primal solution" of the GEM problem (equilibrium chemical speciation), the IPM algorithm always calculates a "dual solution"  - chemical potentials of stoichiometry units (elements) at equilibrium state. By comparing primal and dual solutions, it becomes possible to retrieve pe (Eh), pH, fugacities of gases, very low activities or concentrations, as well as to estimate unknown thermodynamic properties for end-members in solid solution - aqueous solution systems.

Overall, using the GEMS code, you will have the power of advanced chemical thermodynamic modelling at your fingertips, and will eventually achieve an unmatched efficiency in interpreting even complex subsurface- or hydrothermal aquatic systems. Of course, you will need to sacrifice some time and effort required for learning new concepts and some training. To make it easier, we strive to provide a set of test problems and tutorials in the forthcoming releases.
 

What is GEM-Selektor not?

GEMS can do a lot of things for you, but it cannot be everything for everyone.

GEMS cannot replace knowledge of physical chemistry.

It's hardly possible to set up meaningful problems in geochemical thermodynamic modelling and interprete results without a good basic understanding of aquatic chemistry and chemical thermodynamics, even though the underlying concepts are very general and quite simple. Taking a course in the above subjects and/or reading a few good textbooks such as (Anderson and Crerar, 1993; Nordstrom and Munos, 1994; Stumm and Morgan, 1996) would be therefore a valuable investment, helping you to get maximum from the usage of GEMS package at later times.
GEMS is not a substitute for creative research.
Geochemical thermodynamic modelling (as almost any modelling) should be regarded as nothing else than a tool  which helps you in interpreting your geochemical data and in reaching a deeper understanding of how the chemical system works. As any tool, GEMS can be used either in a creative or in a destructive way. 
GEMS cannot formulate chemical modelling problems and interprete results.
Very often, first GEMS runs on your model yield rather unexpected results, telling you that some of your concepts have to be reconsidered. This is normal; a few more modeling iterations will soon give you much more confidence in your data and model predictions. What really makes no sense, is to start modeling something just having the raw data, without a working hypothesis to be verified; clearly, no interpretation of results is possible in this case. GEMS is an instrument for making thermodynamic data dance to your tune, but you are the composer and the piper. No instrument plays itself or writes its own music. GEMS is just a tool for checking how good your ideas are.
GEMS cannot solve for processes in far-from-equilibrium or transient systems.
Some natural, biological or technological systems, especially at short time scales, exist far from equilibrium or in a transient state. In such cases, fundamental assumptions about local and partial equilibria cannot be applied and, therefore, methods of equilibrium chemical thermodynamic cannot be used. Think hard, before doing any model calculations with GEMS or another speciation code, whether the concept of local/partial equilibrium is adequate for your system of interest; otherwise you may just spend time for nothing (and this process is irreversible!).
GEMS is not a tool for plotting phase diagrams.
Plotting phase diagrams is a popular exercise in teaching chemical thermodynamics in petrology or metallurgy, and in aquatic chemistry as well. This requires very serious simplification of the chemical system (to make it observable in 2D or 3D coordinate space), often done using ad hoc  assumptions. However, the GEM modeling is aimed at quantifying  effects of equilibration at given (sequence of) T,P, and bulk composition in full complexity of phases and species in the system, perhaps with many additional kinetic, metastability and/or dispersity constraints. When phases, species and reactions relevant for the particular system's behavior will be identified from the GEMS output, many kinds of diagrams can be plotted using conventional techniques, but now with much less assumptions. In many cases, GEMS batch calculations can be designed in a way that directly output phase or activity diagrams.
GEMS cannot create thermodynamic data or assess their quality.
Any chain is as strong as it's weekest link... Likewise, any modeling result is not better than the input compositional and thermodynamic data. In the past 20 years, a considerable progress has been made in compilation of internally consistent thermodynamic databases, including that supplied as the default database with GEMS code. However, no database covers everything, it is usually necessary to extend it with the standard-state data for some missing relevant species or minerals. GEMS provides a number of advanced tools for extension of the project thermodynamic database, but only you can critically analyze the literature or your own experimental data, and only you can evaluate quality of thus obtained standard-state constants. GEMS cannot replace an expert in critical compilation, matching and retrieval of thermodynamic data, although it offers helpful many tools for such experts.

Some Practical Issues

In most geochemical modelling studies (except maybe simple test examples) you will need to extend thermodynamic database with some new relevant species and phases, or change some thermodynamic properties in a consistent way. A modelling project concept implemented in GEMS lets you performing such data base extensions/modifications conveniently and continue modelling runs right away: already created records with chemical equilibria will be extended automatically. 

All input data and results of GEM calculations are automatically stored in the modelling project data base files and nowhere else. You can easily zip your modelling project directory to back-up your work or to share it with your colleagues. Anyone who has GEMS installed on her/his PC can unpack your project and run it, as you would continue running it on your PC.  

GEMS has a unique structure of thermodynamic database: the standard state data for species are kept either in thermochemical (optionally equation-of-state) format, or as reactions defining standard-state properties of a given species via that of the other species and the reaction. Data of both formats, when entered and saved into database records, can be immediately used in calculations of chemical equilibria. Before saving records, many data consistency checks can be performed.

Unlike most other, command-driven or batch-input-file geochemical modelling codes (e.g., PHREEQC, EQ3/6, GWB, ChemGeo, Selektor-C), GEMS is an interactive modular package in which computational modules share data objects in memory, manipulated by the data base maintenance module. GEMS is written in C/C++ and has a modern, window-based, mouse-menu-driven GUI (graphical user's interface), based on the Qt3.x GUI Toolkit. This toolkit also provides GEMS with cross-platform portability: the source code of GEMS can be compiled and executed on most Intel-based Linux/Unix and Windows platforms.  

In GEMS, no batch files with input data are necessary. The program operation is fully interactive, controlled (as most modern software) by mouse clicking on icons or menu items, selecting record keys from lists, or entering data directly into spreadsheet-like screen forms. Your entered or calculated data is automatically stored in the project data base, where it is kept accessible for subsequent calculations or data sampling, plotting or exporting. This interactive functionality makes the geochemical modeling process more efficient than it has ever been possible before.

Almost all kinds of data accesible in GEMS windows using the data object labels can be printed into ASCII text files using the simple print formatting scripts. A collection of template printing scripts is provided for the user' convenience; all scripts can easily be customized and saved into your project data base.

GEMS GUI provides you with three levels of a context-sensitive hypertext on-line help. For  experienced users, the tooltips (popping up on almost every button or data field) are usually sufficient. Direct access to all pages of the HTML documentation is possible from any dialog, screen form, and even a single data field cell on any screen form. If you have a fast internet connection (LAN, ADSL, cable etc.) then you can set up GEMS to get help pages on-line from the GEMS home site at LES PSI: in this way you can get the last-updated documentation before it will be included in the next release. 

GEMS numerical kernel is an Interior Points Method (IPM) module for non-linear minimization of total Gibbs energy of the heterogeneous multi-phase chemical system (Karpov, Chudnenko and Kulik 1997, Amer.J.Sci. v.297, p.767-806). Since GEMS-PSI version 1.94a, this kernel comes optimized for the mass balance precision, convergence, and extended input data diagnostics as an  enhanced GEM IPM-2 module  to meet needs of the most demanding modelling applications related to performance assessment in nuclear waste disposal or in aquatic geochemistry.

The GEM IPM-2 numerical module is fast and robust, and it can check internal consistency of the input data. There are important classes of problems, such as redox equilibrium in solid solution-aqueous solution systems, which are difficult to solve using the commonly available speciation codes (such as PHREEQC) based on the law-of-mass action algorithm and reactions logK database. However, such problems are easily and directly solved with the IPM - even in the systems including several (non-ideal) multi-component solid solutions or multi-surface sorption phases.

Of course, GEMS solves traditional equilibrium aqueous speciation problems, perhaps, somewhat slower than the LMA code, but providing even more output parameters at less input data. pH, Eh, ionic strength and fugacities of gases are always obtained as  output (not input) parameters in GEM calculations. This helps greatly in investigating behaviour of redox-dependent systems.

Scientific Basis

GEMS is an implementation of a convex programming approach to Gibbs energy minimization calculations of chemical equilibria ([1981KAR]; [1997KAR/CHU]; [2000KUL]; [2001KAR/CHU]; [2002KUL]). More details about the scientific side can be found in theorgem.html manual or GEMS-PSI home page (http://les.web.psi.ch/Software/GEMS-PSI). The numerical core of GEMS is an Interior Points Method (IPM) non-linear minimization module, recently improved for much better accuracy and precision (IPM-2 module).
